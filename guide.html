Release gate

Before production traffic.

Use these checks for automated tests, integration rehearsals, and final transaction review.

  • Payload contains exactly the field set expected for its operation.
  • hash is a valid transaction hash for every non-deploy action.
  • tokenid base64-decodes and normalizes to the expected label.
  • Recipient appears in the deployment recipient policy where required.
  • Duration, coefficient, signature, and image rules match the current indexer version.
  • Owner and expiry state match after confirmation.
Operational cautionSRC-101 has strict, revision-sensitive field validation. Generate payloads from the active parser contract and regression-test them before using real names.
